ME 136

Flight controls, estimation, and hover logic built from first principles.

ME 136 became a full-stack controls project for me: I moved from direct quadrotor motor actuation and sensor inspection into estimation, control allocation, and autonomous hover behavior using real onboard measurements.

Why It Matters

A recruiter-ready view of the full lab arc.

Rather than treating each lab as an isolated exercise, this page frames the course as one continuous controls and embedded-systems build. The work progressed from actuator command generation to sensor-based estimation, then into closed-loop attitude control and hover stabilization using fused measurements from the IMU, height sensor, and optical-flow sensor.

Along the way, I wrote embedded C++ control logic, used MATLAB to analyze logged telemetry, tuned time constants and controller gains, and validated how estimation and control performance changed across experiments.

Technical Summary

What I learned and implemented across the lab sequence

In the first labs, I worked directly with the quadrotor's motor commands and IMU signals, using embedded C++ to command all four motors and inspect accelerometer and gyro measurements. That groundwork established a practical understanding of how low-level inputs, raw measurements, and telemetry outputs interacted inside the real-time flight loop.

I then moved into actuator modeling by replacing direct motor commands with a speed-based interface using the provided conversion utilities. From there, the course shifted into estimation: I implemented roll, pitch, and yaw estimation by correcting gyroscope drift with a complementary-filter structure that blended integrated rate gyro data with accelerometer-based attitude measurements.

Once attitude estimation was in place, I developed a closed-loop attitude controller for the quadrotor. That work included computing desired body rates from attitude error, shaping rate dynamics with time constants, and using a mixer matrix to translate total thrust and desired torques into four motor force commands. This tied together rigid-body dynamics, controller design, and embedded implementation.

The later labs extended the stack into position-related estimation and hover behavior. I estimated height and vertical velocity from the range sensor while correcting for vehicle attitude, then used optical-flow data with gyroscope compensation to estimate horizontal velocity. With those states available, I implemented horizontal damping logic, vertical second-order height control, and the corresponding roll, pitch, and thrust commands needed to maintain stable hover near a desired altitude.

Throughout the sequence, I used MATLAB to process logged telemetry, plot estimated states and commands over time, compare desired versus measured behavior, and evaluate how tuning parameters affected stability, damping, and overall flight response. The result was a course-long progression through embedded controls, sensor fusion, dynamic modeling, and experimental validation on a quadrotor platform.
